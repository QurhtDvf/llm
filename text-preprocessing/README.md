# 分かち書きの歴史とLLM前処理

### Research Notes

本ノートは、自然言語処理における分かち書き（word
segmentation）の歴史を、特に大規模言語モデル（LLM）の前処理との関係から整理した研究メモである。\
分かち書きは単なる文字列処理ではなく、語彙設計・計算量・モデル効率に直結する問題であり、NLPの歴史の中で役割を変化させてきた。

本資料では以下を整理する。

-   分かち書き技術の歴史
-   サブワードトークナイザーのアルゴリズム
-   LLMにおけるトークン設計との関係

------------------------------------------------------------------------

# 1. 連続書字から語境界へ

古代ギリシア語やラテン語の文書では、単語の区切りを持たない **scriptio
continua（連続書字）** が一般的であった。

例

    THISISANEXAMPLEOFSCRIPTIOCONTINUA

読者は文脈から語境界を推定していた。

中世ヨーロッパにおいて読解効率向上のために空白が導入され、英語などの西欧言語では

    This is an example

のように **空白＝語境界** という規則が成立した。

この慣習は計算機処理において極めて重要であり、英語の初期NLPでは

    tokens = text.split(" ")

という単純な処理でトークン化が可能であった。

------------------------------------------------------------------------

# 2. 空白を持たない言語と形態素解析

日本語・中国語・タイ語などでは語間空白が存在しないため、語境界推定アルゴリズムが必要となる。

これが **形態素解析 (morphological analysis)** である。

典型的手法

1.  辞書から単語候補を列挙
2.  文全体の確率を計算
3.  最適経路を選択

多くの場合

    W* = argmax P(W)

として定式化される。

代表的アルゴリズム

-   Viterbi
-   HMM
-   CRF

代表的ツール

-   MeCab
-   Juman
-   ChaSen

処理パイプライン

    text
    ↓
    morphological analysis
    ↓
    word sequence
    ↓
    statistical model

------------------------------------------------------------------------

# 3. 統計言語モデルとサブワード

N-gram言語モデルの時代になると、単語単位の問題が顕在化した。

主な問題

-   語彙爆発
-   未知語問題

語彙サイズ

    V ≈ 100k - 1M

これを回避するため **subword unit** の概念が導入された。

------------------------------------------------------------------------

# 4. Byte Pair Encoding (BPE)

BPEは元々データ圧縮アルゴリズムであり、ニューラル機械翻訳でトークナイザーとして採用された。

## アルゴリズム

初期状態

    l o w e s t
    l o w e r
    n e w e s t

手順

1.  文字を初期語彙とする
2.  最頻出ペアを探索
3.  ペアを結合
4.  新しいトークンとして追加
5.  繰り返し

例

    low + er
    low + est

擬似コード

    vocab = characters

    while vocab_size < target:
        pair = most_frequent_pair(corpus)
        merge(pair)

特徴

-   語彙サイズ制御
-   未知語処理可能
-   実装が簡単

------------------------------------------------------------------------

# 5. SentencePiece

SentencePieceはGoogleによって提案されたトークナイザーである。

特徴

-   空白を前提としない
-   raw text から直接学習
-   多言語対応

従来

    text → words → subwords

SentencePiece

    text → subwords

------------------------------------------------------------------------

## 5.1 Unigram Language Model

SentencePieceでは **Unigram LM** が利用されることが多い。

目的

    W* = argmax P(W)

ここで

    P(W) = ∏ P(w_i)

語彙候補集合から確率的に最適分割を選択する。

学習手順

1.  大きな語彙集合を初期化
2.  EMアルゴリズムで確率推定
3.  低確率トークン削除
4.  語彙収束

------------------------------------------------------------------------

# 6. Transformer時代

Transformer以降のLLMではサブワードトークンが標準となった。

  Model   Tokenizer
  ------- ---------------
  BERT    WordPiece
  GPT     BPE
  T5      SentencePiece
  LLaMA   SentencePiece

この段階で分かち書きは

**言語学的処理 → 計算最適化問題**

へ変化した。

------------------------------------------------------------------------

# 7. LLMトークン最適化

LLMの計算コストはトークン数に依存する。

Transformer計算量

    O(n²)

n = トークン数

したがって

-   トークン数削減
-   情報密度最大化

が重要となる。

------------------------------------------------------------------------

## 7.1 語彙サイズ

embedding行列

    E ∈ R^{V × d}

V = vocabulary size

通常

    V ≈ 32k – 100k

------------------------------------------------------------------------

## 7.2 トークン設計のトレードオフ

  要素         影響
  ------------ ----------------
  大語彙       トークン数減少
  小語彙       未知語耐性
  長トークン   attention削減
  短トークン   汎用性向上

最適化問題

    minimize compute_cost
    subject to model_accuracy

------------------------------------------------------------------------

# 8. 最近の研究

近年の研究ではトークナイザー自体を廃止する方向も存在する。

主なアプローチ

-   byte-level models
-   character transformers
-   tokenizer-free LLM

ただし計算効率の観点から、現状ではサブワードトークンが最も実用的とされている。

------------------------------------------------------------------------

# 9. まとめ

分かち書き技術の進化

  時代        技術
  ----------- -------------------------
  古典        scriptio continua
  初期NLP     空白分割
  日本語NLP   形態素解析
  NMT         BPE
  LLM         SentencePiece / subword

現在のLLMにおいて分かち書きは

-   語彙設計
-   推論コスト
-   モデル効率

に直結する **重要なアーキテクチャ要素** となっている。
