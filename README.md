# 大規模言語モデル（LLM）の歴史

> 本ノートは、大規模言語モデル（LLM: Large Language Model）の歴史を研究ノート形式で整理したものである。自然言語処理（NLP）の発展は、ルールベース処理・統計モデル・ニューラルネットワーク・Transformerへと段階的に進化してきた。本資料では、LLMの成立までの主要な技術的転換点を時系列で整理し、各時代の背景・課題・意義を詳述する。

## 1. 初期の自然言語処理（1950s–1980s）

### 背景

コンピュータが登場した直後から、人間の言語をいかに機械で処理するかという問いが研究者を惹きつけてきた。1950年にアラン・チューリングが提唱した「チューリングテスト」は、機械が人間と区別のつかない会話ができるかを問うものであり、NLP研究の精神的な出発点となった。

### 主なアプローチ

**ルールベース言語処理**
人間が手作業で文法規則を記述し、それに基づいて構文解析や翻訳を行う手法。1966年に開発された対話システム **ELIZA**（MIT）は、パターンマッチングによって心理療法士の会話を模倣した初期の成果である。

**統計言語モデル（初期）**
言語を確率モデルとして定式化し、単語列の同時確率を次のように表現する。

```
P(w1, w2, ..., wn)
```

### 時代の特徴と限界

| 特徴 | 限界 |
|------|------|
| 手作業の文法ルール | スケールしない（言語の多様性に対応困難） |
| 辞書ベース処理 | 未知語・新語への対応が困難 |
| 小規模コーパス | 統計的な汎化能力がない |

## 2. N-gram言語モデル（1990s）

### 概要

1990年代になると、大規模コーパスを活用した**統計的言語モデル**が主流となった。音声認識・機械翻訳の精度向上に大きく貢献し、IBMやAT&Tベル研究所が研究をリードした。

### モデルの定式化

N-gramモデルでは、**マルコフ近似**（直前のN-1語にのみ依存するという仮定）に基づき、条件付き確率を次のように近似する。

```
P(wi | w1, ..., w_{i-1}) ≈ P(wi | w_{i-n+1}, ..., w_{i-1})
```

- **ユニグラム (N=1):** 各単語を独立に扱う
- **バイグラム (N=2):** 直前の1語に依存
- **トライグラム (N=3):** 直前の2語に依存（実用上よく用いられた）

### スムージング技術

学習データに出現しない語のN-gramに対して確率ゼロを防ぐため、様々なスムージング手法が開発された。

- **ラプラス（加算）スムージング:** 全カウントに一定値を加算
- **Katz バックオフ:** 高次N-gramのカウントが少ない場合に低次N-gramへ後退
- **Kneser-Ney スムージング:** 単語の出現多様性を考慮した高精度な手法

### 残された課題

- **長距離依存の無視:** N-gramは限られた文脈しか参照できない
- **データスパース問題:** Nが大きくなるほど学習データが指数的に必要
- **意味の欠如:** 構造的に似た文でも意味が無関係な場合を区別できない

## 3. ニューラル言語モデル（2000s）

### Bengio et al. (2003) の提案

2003年にYoshua Bengioらが発表した論文 *"A Neural Probabilistic Language Model"* により、ニューラルネットワークを用いた言語モデルが初めて本格的に提案された。この研究が後のディープラーニング時代のNLPの礎となる。

### 基本構造

```
入力 (one-hot) → 埋め込み層 (embedding) → 隠れ層 (tanh) → 出力層 (softmax)
```

単語を低次元の実数ベクトルとして表現する**単語埋め込み（Word Embedding）** の概念がここで初めて導入された。

### 意義

| 従来手法（N-gram）との比較 | 詳細 |
|---------------------------|------|
| 分散表現 | 単語を連続ベクトルで表現することで類似語を近くに配置できる |
| 汎化能力の向上 | 未見の単語列に対しても類似した語順から確率を推定できる |
| 意味空間の獲得 | 「王」と「女王」のように意味的に近い単語が近接するベクトル空間が形成される |

### 課題

計算コストが高く、当時のハードウェア（CPU）では大規模データへの適用が困難だった。GPUを用いた並列計算の普及が、このアプローチの本格的な実用化を後押しすることになる。

## 4. Word Embedding（2013）

### Word2Vecの登場

2013年にGoogleのTomas Mikolovらが発表した **Word2Vec** は、単語の分散表現学習を劇的に効率化し、NLP研究に革命をもたらした。

### 2つのアーキテクチャ

**CBOW (Continuous Bag-of-Words)**
周囲の文脈語から中心語を予測する。

```
文脈語群 → 平均化 → 中心語を予測
maximize log P(word | context)
```

**Skip-gram**
中心語から周囲の文脈語を予測する。少ないデータでも高品質な表現が得られ、特に稀少語に強い。

```
中心語 → 文脈語群を予測
maximize log P(context | word)
```

### 線形代数的な意味演算

Word2Vecが示した最も驚くべき性質は、意味関係がベクトル演算で表現できることだった。

```
king − man + woman ≈ queen
Paris − France + Japan ≈ Tokyo
```

### 後継手法

- **GloVe（2014, Stanford）:** 単語の共起行列を行列分解し、コーパス全体の統計を利用
- **fastText（2016, Facebook）:** 単語をサブワード（文字N-gram）に分解し、未知語対応を強化

## 5. RNN / LSTM時代（2014–2017）

### Recurrent Neural Networkの導入

単語が順序を持つシーケンスである言語に対し、RNNは隠れ状態を通じて過去の文脈を引き継ぎながら逐次処理を行う。

```
h_t = tanh(W_h * h_{t-1} + W_x * x_t + b)
```

### LSTMとGRUによる長期記憶の実現

基本的なRNNは**勾配消失問題**により長距離の依存関係を学習できなかった。これを解決したのが **LSTM（Long Short-Term Memory, Hochreiter & Schmidhuber 1997）** である。

LSTMは**入力・忘却・出力ゲート**という3つの制御機構によって、重要な情報を長期保持し不要な情報を忘れることができる。

```
セル状態 (cell state) = 長期記憶
隠れ状態 (hidden state) = 短期記憶
```

**GRU（Gated Recurrent Unit, 2014）** はLSTMを簡略化したモデルで、パラメータ数を削減しつつ同等の性能を発揮する。

### Seq2Seq と Attention 機構の萌芽

2014年にGoogleが提案した **Seq2Seq（Encoder-Decoder）** モデルは機械翻訳に革命をもたらした。さらに2015年にBahdanauらが提案した **Attention機構** により、デコーダが入力系列の任意の位置に注目できるようになり、長文の翻訳品質が大幅に向上した。このAttentionの概念が後のTransformerへと発展する。

### 課題

- **並列化困難:** 逐次処理のため、長いシーケンスの学習に多大な時間がかかる
- **長距離依存の限界:** LSTMでも数百トークンを超える依存関係の学習は困難
- **勾配問題:** 非常に深いネットワークでは依然として学習が不安定

## 6. Transformerの登場（2017）

### "Attention Is All You Need"

2017年にGoogleの研究チームが発表した論文 **"Attention Is All You Need"**（Vaswani et al.）は、RNNを一切用いない純粋なAttentionベースのアーキテクチャ「Transformer」を提案した。これはNLP史上最も影響力の大きい論文の一つとなる。

### Self-Attention 機構

入力系列の各トークンが、同じ系列内の他の全トークンとの関連度を直接計算する。

```
Attention(Q, K, V) = softmax(Q Kᵀ / √d_k) V
```

- **Q (Query):** 注目したいトークンの表現
- **K (Key):** 注目される側のトークンの表現
- **V (Value):** 実際に受け取る情報
- **√d_k:** スケーリング係数（内積が大きくなりすぎるのを防ぐ）

### Multi-Head Attention

複数の注意ヘッドを並列に走らせることで、異なる種類の関係（構文・意味・参照関係など）を同時に学習する。

```
MultiHead(Q,K,V) = Concat(head_1, ..., head_h) W^O
head_i = Attention(Q W_i^Q, K W_i^K, V W_i^V)
```

### Transformerのアーキテクチャ

```
Encoder: Multi-Head Self-Attention → Add & Norm → Feed-Forward → Add & Norm
Decoder: Masked Self-Attention → Cross-Attention (Enc-Dec) → Feed-Forward
```

### Transformerが解決した問題

| 課題 | 解決方法 |
|------|---------|
| 並列化困難（RNNの逐次処理） | 全トークンを同時に処理可能 |
| 長距離依存 | 任意の2トークン間を直接接続（O(1)パス長） |
| スケーラビリティ | モデルサイズを大規模化しやすい構造 |

## 7. 事前学習モデル（2018–2020）

### Transfer Learning の確立

Transformerの登場以降、**大規模コーパスで事前学習し、タスクごとにファインチューニングする**というパラダイムが確立された。

```
大規模コーパス → 事前学習 (pretraining) → タスク固有データ → ファインチューニング (finetuning)
```

### 代表的モデルの比較

| モデル | 開発元 | 年 | 構造 | 事前学習目標 | 特徴 |
|--------|--------|-----|------|-------------|------|
| ELMo | AllenAI | 2018 | 双方向LSTM | 言語モデル | 文脈依存の単語表現 |
| BERT | Google | 2018 | Transformerエンコーダ | MLM + NSP | 双方向文脈、理解タスクに強い |
| GPT | OpenAI | 2018 | Transformerデコーダ | 自己回帰LM | 左→右の自己回帰、生成に適する |
| XLNet | Google/CMU | 2019 | Transformer-XL | Permutation LM | BERTとGPTの利点を統合 |
| RoBERTa | Facebook | 2019 | BERTベース | MLM（改良版） | BERTの学習条件を徹底改善 |
| T5 | Google | 2020 | Encoder-Decoder | Text-to-Text | 全タスクをテキスト変換として統一 |

### BERT の詳細

BERTは以下の2つの事前学習目標を採用した。

- **MLM（Masked Language Model）:** 入力の15%をマスクし、その語を予測する
- **NSP（Next Sentence Prediction）:** 2文が連続するかどうかを予測する（後の研究でNSPは不要とも判明）

### GPT の詳細

GPTは自己回帰モデルであり、左から右へと次のトークンを順番に予測する。

```
P(w_t | w_1, ..., w_{t-1})
```

この設計がテキスト生成に非常に適しており、GPT-2以降の「大規模生成モデル」の方向性を決定づけた。

## 8. GPTシリーズとLLMの成立

### GPT系列の進化

| モデル | 年 | パラメータ数 | 主な特徴 |
|--------|-----|-------------|---------|
| GPT | 2018 | 117M | Transformerデコーダの基礎実証 |
| GPT-2 | 2019 | 1.5B | Zero-shotで多様なタスクをこなすことを初めて示した |
| GPT-3 | 2020 | 175B | Few-shotプロンプティングで多くのNLPタスクでSOTA |
| InstructGPT | 2022 | 175B | RLHFによる指示追従能力の大幅向上 |
| GPT-4 | 2023 | 非公開（推定1T超） | マルチモーダル対応、推論能力の飛躍的向上 |

### GPT-3 の衝撃

GPT-3は**In-Context Learning（文脈内学習）** と呼ばれる能力を示した。モデルのパラメータを更新することなく、プロンプト中にいくつかの例（few-shot examples）を与えるだけで未知のタスクに対応できることを実証した。

```
プロンプト例（few-shot）:
  英語→フランス語: "sea otter" → "loutre de mer"
  英語→フランス語: "cheese" → "fromage"
  英語→フランス語: "car" → ?  (モデルが "voiture" を生成)
```

### RLHFによる指示追従

InstructGPT（2022）で採用された **RLHF（Reinforcement Learning from Human Feedback）** は、人間の評価者によるフィードバックを報酬信号として強化学習を行い、モデルを人間の意図に沿った挙動に誘導する手法である。これにより、有害な出力の削減と指示追従能力が大幅に向上した。

## 9. Scaling Law

### Kaplan et al. (2020)

OpenAIの研究チームは2020年の論文 *"Scaling Laws for Neural Language Models"* において、言語モデルの損失が以下の3要素に対してべき乗則でスケールすることを発見した。

```
Loss ≈ C · N^{-α}   (モデルパラメータ数 N に対して)
Loss ≈ C · D^{-β}   (データ量 D に対して)
Loss ≈ C · F^{-γ}   (計算量 F に対して)
```

### Chinchilla の法則（2022）

DeepMindの研究チームはその後、**Chinchillaモデル** の実験により、最適な学習には「モデルサイズに比例したデータ量」が必要であることを示した。具体的には、パラメータ数1Bにつきおよそ20Bトークンのデータが最適とされる。

```
最適データ量 ≈ 20 × モデルパラメータ数
```

これにより、同じ計算コストなら「大きなモデルを少ないデータで学習する」よりも「適切サイズのモデルを多くのデータで学習する」方が効率的であることが示された。

### Scaling Law の意義

- 事前に実験コストを予測できるため、大規模モデル開発が計画的に行えるようになった
- モデルサイズの肥大化競争を促し、GPT-3 → PaLM (540B) → Gemini Ultra といった超大規模モデルの開発へとつながった
- 一方で、スケールだけでは解決できない推論能力の限界（算数・論理）も明らかになり、**Chain-of-Thought (CoT)** などの手法の研究が促進された

## 10. LLMエコシステム（2023–）

### 主要技術の拡張

現在のLLMは単体モデルではなく、多様な周辺技術と組み合わさってエコシステムとして発展している。

| 技術 | 概要 | 主な用途 |
|------|------|---------|
| RLHF | 人間フィードバックによる強化学習 | 安全性・指示追従の向上 |
| RAG（Retrieval-Augmented Generation） | 外部知識ベースを検索して文脈に組み込む | 事実精度の向上、知識の最新化 |
| Tool Use / Function Calling | APIや計算機などの外部ツールを呼び出す | 計算・検索・コード実行 |
| Agent Systems | 複数ステップの計画・実行を自律的に行う | 複雑タスクの自動化 |
| CoT（Chain-of-Thought） | 中間推論ステップを明示させる | 数学・論理推論の精度向上 |

### オープンソースモデルの台頭

- **LLaMA（Meta, 2023）:** 研究向けに公開されたオープンウェイトモデル。コミュニティによる派生モデル開発を促進
- **Mistral / Mixtral:** 高効率なアーキテクチャ（MoE: Mixture of Experts）を採用し、小規模なパラメータで大規模モデルに匹敵する性能を実現

### 推論最適化技術

LLMの推論（inference）コストを削減するための技術が急速に発展している。

- **KVキャッシュ:** 過去の計算済みキー・バリューを再利用し、逐次生成を高速化
- **量子化（Quantization）:** 重みをFP16/INT8/INT4に圧縮してメモリ使用量を削減
- **Flash Attention:** メモリ効率的なAttention計算で速度と省メモリを両立
- **投機的デコーディング（Speculative Decoding）:** 小型モデルで候補を先読みし大型モデルで検証することで生成速度を向上

### マルチモーダルへの拡張

テキストのみに留まらず、画像・音声・動画・コードなど複数のモダリティを統合したモデルが登場している（GPT-4V, Gemini Ultra, Claude 3など）。

## 11. まとめ

### LLMの歴史的変遷

| 時代 | 主要技術 | 代表的手法・モデル | 解決した課題 |
|------|---------|-------------------|-------------|
| 初期NLP（〜1980s） | ルールベース | ELIZA, 文法規則 | 言語の形式的処理 |
| 統計NLP（1990s） | N-gram | バイグラム, トライグラム | 大規模コーパスの活用 |
| ニューラルNLP（2000s〜） | 分散表現 | Word2Vec, GloVe | 意味の連続空間表現 |
| Seq2Seq + Attention（2014〜） | LSTM + Attention | Seq2Seq, Bahdanau Attention | 長距離依存・翻訳 |
| Transformer（2017〜） | Self-Attention | BERT, GPT | 並列化・スケーラビリティ |
| 大規模事前学習（2020〜） | Transfer Learning | GPT-3, PaLM, LLaMA | 汎用言語能力・Few-shot学習 |
| LLMエコシステム（2023〜） | RLHF, RAG, Agent | GPT-4, Gemini, Claude | 安全性・信頼性・実用化 |

### 現在の研究フロンティア

- **推論能力の向上:** Chain-of-Thought、Self-Consistency、o1系モデルによるTest-time Compute
- **効率化:** より少ないパラメータ・データで高性能を達成するアーキテクチャ探索
- **マルチモーダル:** テキスト・画像・音声・動画の統合理解と生成
- **エージェント化:** LLMをコアとした自律的問題解決システムの構築
- **安全性と整合性（Alignment）:** 人間の価値観に沿った安全なLLMの実現
