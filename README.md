# LLMの歴史

### Research Notes

本ノートは、大規模言語モデル（LLM: Large Language
Model）の歴史を研究ノート形式で整理したものである。\
自然言語処理（NLP）の発展は、統計モデル、ニューラルネットワーク、そしてTransformerへと進化してきた。本資料では、LLMの成立までの主要な技術的転換点を時系列で整理する。

------------------------------------------------------------------------

# 1. 初期の自然言語処理（1950s--1980s）

初期のNLPは主に以下の二つの方向で研究されていた。

-   ルールベース言語処理
-   統計言語モデル

代表例としては機械翻訳研究がある。

この時代の特徴

-   手作業の文法ルール
-   辞書ベース処理
-   小規模コーパス

言語モデルは確率モデルとして定式化され、基本問題は次のように表現される。

    P(w1, w2, ..., wn)

------------------------------------------------------------------------

# 2. N‑gram言語モデル（1990s）

1990年代になると、大規模コーパスを利用した統計的言語モデルが主流となった。

N‑gramモデルでは

    P(w_i | w_{i-n+1}, ..., w_{i-1})

の形で確率を近似する。

特徴

-   Markov近似
-   大規模コーパス利用
-   未知語問題

しかし

-   長距離依存
-   文脈理解

などの問題が残った。

------------------------------------------------------------------------

# 3. ニューラル言語モデル（2000s）

2003年にニューラル言語モデルが提案され、単語埋め込み（word
embedding）が導入された。

基本構造

    one-hot → embedding → neural network → softmax

この段階で

-   分散表現
-   意味空間

という概念が広く普及した。

------------------------------------------------------------------------

# 4. Word Embedding（2013）

2013年にWord2Vecが登場し、単語ベクトルがNLPの中心技術となった。

代表的モデル

-   CBOW
-   Skip‑gram

目的関数

    maximize log P(context | word)

これにより意味的類似性がベクトル空間で表現されるようになった。

例

    king − man + woman ≈ queen

------------------------------------------------------------------------

# 5. RNN / LSTM時代（2014--2017）

Recurrent Neural
Networkが言語モデルに導入され、長距離依存を扱えるようになった。

代表モデル

-   RNN
-   LSTM
-   GRU

特徴

-   文脈情報を状態として保持
-   シーケンス処理

しかし

-   並列化困難
-   長文処理の限界

という問題が残った。

------------------------------------------------------------------------

# 6. Transformerの登場（2017）

2017年の論文

    Attention Is All You Need

によりTransformerアーキテクチャが提案された。

基本構造

    self-attention
    +
    feed-forward network

自己注意機構

    Attention(Q,K,V) = softmax(QKᵀ / √d)V

Transformerの利点

-   並列計算可能
-   長距離依存処理
-   スケーラブル

このアーキテクチャが後のLLMの基盤となる。

------------------------------------------------------------------------

# 7. 事前学習モデル（2018--2020）

Transformerを用いた事前学習モデルが登場した。

代表例

  モデル   特徴
  -------- --------------------
  BERT     双方向エンコーダ
  GPT      自己回帰モデル
  T5       テキストtoテキスト

事前学習の考え方

    large corpus
    ↓
    pretraining
    ↓
    finetuning

この枠組みが現在のLLMの基本となる。

------------------------------------------------------------------------

# 8. GPTシリーズとLLMの成立

大規模化によって言語モデルの性能が大きく向上した。

  モデル   年
  -------- ------
  GPT      2018
  GPT‑2    2019
  GPT‑3    2020
  GPT‑4    2023

特徴

-   数十億〜数千億パラメータ
-   大規模コーパス
-   汎用言語能力

------------------------------------------------------------------------

# 9. Scaling Law

言語モデル性能は

-   モデルサイズ
-   データ量
-   計算量

の関数としてスケールすることが知られている。

概念的には

    Loss ≈ f(model_size, data, compute)

と表される。

この発見により、大規模モデル開発が加速した。

------------------------------------------------------------------------

# 10. LLMエコシステム（2023--）

現在のLLMは単体モデルではなく、周辺技術を含むエコシステムとして発展している。

主要技術

-   RLHF
-   RAG
-   Tool use
-   Agent systems

さらに

-   推論最適化
-   KVキャッシュ
-   分散推論

などのインフラ技術も重要になっている。

------------------------------------------------------------------------

# まとめ

LLMの歴史は大きく以下の段階に整理できる。

  時代            技術
  --------------- -----------------
  初期NLP         ルールベース
  統計NLP         N‑gram
  ニューラルNLP   RNN / Embedding
  Transformer     Attention
  LLM             大規模事前学習

現在のLLM研究は

-   モデルスケーリング
-   推論最適化
-   エージェント化

という方向に進んでいる。
